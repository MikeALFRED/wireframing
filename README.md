**Wireframes** are simple sketches of a website - like a blueprint for a house. They show:

- Where things go (buttons, text, images)
- How pages connect
- What the website will do

**Why we use them:**
- Plan the layout quickly
- Easy to make changes
- Everyone understands the plan

## ðŸ“‹ Wireframe Elements

### 1. Layout Structure
**What it is:** How the page is organized into sections
**Example:** Header on top, search bar below, property grid in middle
**Why it matters:** Creates a clean, easy-to-use layout

### 2. Navigation  
**What it is:** How users move between pages
**Example:** Menu bar, back buttons, breadcrumbs
**Why it matters:** Helps users find what they need quickly

### 3. Content Placement
**What it is:** Where text, images, and information appear
**Example:** Property photos at top, details below, booking box on side
**Why it matters:** Shows most important information first

### 4. Functionality
**What it is:** What users can do on each page
**Example:** Search filters, date pickers, booking buttons
**Why it matters:** Makes the website interactive and useful
